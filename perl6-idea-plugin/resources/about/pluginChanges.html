Change List

2018.12.0

  • Improve performance of editing larger files.
  • Pressing Enter inside of [] square brackets now creates indent the same way as () and {} do.
  • Tests can be run in parallel now. The number of tests to run at the same time defaults to 1 and can be changed using the Run configuration editor.
  • Improve code formatter to handle blocks, hash literals, array literals and spaces around semicolon better.
  • Make rename refactoring in-place for routine parameters and local variables declared with state and my declarators.
  • Make rename refactoring work on module names. Applying rename refactoring invoked from local module usage or renaming a Perl 6 module file will result in appropriate file system changes (renaming from Foo to Foo::Bar will create the new directory if needed, renaming from Foo::Bar to Foo will remove one if no other modules use that prefix). All references in require and use statements will be updated.
  • Add the Extract Variable and Extract Constant refactoring, which can extract both expressions and statements. It supports choosing of the appropriate sub-expression, in-place renaming of the new variable or constant introduced, and replacing of existing occurrences of the expression usage.
  • Add a quickfix to the unknown private method call annotation, which stubs a private method. Signature parameters of a the new stubbed method are inferred based on call arguments, and can be in-place renamed afterwards.
  • Fix new module creation on Windows.
  • Fix an annotation that has warned about usage of a package when module in in depends section of META6.json had adverbs, for example, false warning on use JSON::Schema when META6.json had JSON::Schema:ver<0.1> in the depends section.
  • Improve an annotation that warns about the use of the does trait with a class: now it supports the does trait applied with the also declarator, and also annotates the trait instead of the package name and works with all types of packages.
  • Complete methods on subset, using its base type provided with the of trait or defaulting to Any.
  • Fix an exception which occurred when the provides section was missing in META6.json file.
  • Fix an exception on completion of a method inside of an anonymous package (class, role, etc).
  • Fix an exception on completion of a module name of require statement.
  • Fix an annotation that has warned about range ending in a prefixed expression as unfinished.
  • Do not annotate usage of late-bound modules.
  • Fix completion inside of an anonymous subroutine.
  • Complete methods of the resulting value type of a subroutine or method call, when the return type is explicitly set in signature.
  • Offer values of enum in scope for auto-completion and resolution of enum value names.
  • Fix potential completion and reference resolution problems with subset types.
  • Fix redundant unit keyword in defined package declaration and missing unit keyword in undefined package declaration annotator.
  • Fix an exception that could occur during internal indexing.
  • Fix an annotation that has warned about absence of an endpoint in a range when range definition was multiline.
  • Fix parser misbehavior that could occur when typing a loop statement in some cases.
