Change List

2018.7.1

  • Add first steps towards method auto-completion, supporting it for method calls on self and type objects; includes methods composed from roles and inherited from base classes, whether they are internal to the project, in the CORE builtins, or from an external module
  • Support Go To Definition (Ctrl+B) for method calls within the project that are made on self or a type object
  • Implement Move Statement support (Ctrl+Shift+Up, Ctrl+Shift+Down), so that multi-line statements can be moved
  • Correct some cases where a sub call imported from another module within the project was wrongly detected as undeclared and failed to Go To Definition
  • Fix a bug causing partially missing auto-complete information from external modules
  • Make the display of test results more incremental, showing them as each file completes its run
  • Add Project view in the standalone Comma IDE
  • Made Project view hide .precomp directories
  • Parse and highlight version literals
  • Parse and highlight macro and quasi
  • Correct whitespace parsing in various places in regexes, so that a backslashed comment character is never incorrectly treated as unspace
  • Avoid a crash when encountering incorrect JSON in the META6.json file
  • Fix a crash that could occur sometimes during project import or creation
  • Correct highlighting of a sub or private method call to indicate other calls to the same sub to put a background highlight on the name, not on the whole argument list
